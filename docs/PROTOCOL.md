# Wire Protocol

All messages between the desktop, the companion, peer desktops and the optional
cloud relay are JSON objects sent over a single WebSocket. The TypeScript
source of truth lives in `src/shared/types.ts` as `WireMessage`.

## Roles

```
companion  → drives a single desktop
peer       → another Presentatool install on the LAN (bidirectional)
relay      → cloud WebSocket that forwards companion ↔ desktop across networks
```

## Handshake

```jsonc
// companion → desktop
{ "kind": "hello", "role": "companion", "token": "<one-time>" }

// desktop → companion
{ "kind": "welcome", "peer": { "id": "...", "name": "Mac Studio", ... } }
{ "kind": "presentations", "list": [ { "id": "...", "title": "...", "kind": "pptx", ... } ] }
```

Tokens are generated by `Server.createPairingToken()`, expire after 5 minutes,
and are single-use.

## Clicker commands

```jsonc
{ "kind": "click", "command": { "type": "next" } }
{ "kind": "click", "command": { "type": "goto", "index": 7 } }
```

The supported `type`s: `next | prev | first | last | blank | goto | start | end`.

## Live slide info

The desktop polls the platform adapter every ~1.5s and broadcasts to all open
remotes:

```jsonc
{ "kind": "slide", "info": { "index": 4, "total": 27, "title": "Why", "notes": "...", "nextTitle": "How" } }
```

## Sync

Each peer announces its content with `sync-offer`. The receiver requests
anything new, and chunks come back as base64. `sync-done` marks completion.

```jsonc
{ "kind": "sync-offer", "presentations": [ { "presentationId": "...", "versionId": "...", "size": 12345, "kind": "pptx", "title": "Q1 Plan" } ] }
{ "kind": "sync-request", "presentationId": "...", "versionId": "..." }
{ "kind": "sync-chunk", "presentationId": "...", "versionId": "...", "index": 0, "total": 4, "data": "..." }
{ "kind": "sync-done", "presentationId": "...", "versionId": "..." }
```
